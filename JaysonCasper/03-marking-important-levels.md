# Marking Important Levels

Source lesson: marking-important-levels
(Short supplemental clarification video on when to add/remove S&R levels — same core rule as the later "Getting Accurate Daily & Weekly Levels" lesson, with a chart-organization tip added.)

## Core rule (reactivity decays with each touch)
- A level is confirmed once you see at least two candle bodies open/close (or repeatedly wick to) the same price — mark it as a new level of that timeframe (daily/weekly/4H/etc.) as soon as it's printed.
- **First touch after formation = most reactive.** Second touch can usually still be expected to react. By the third+ touch, expect progressively weaker reactions — sometimes price just blows straight through with no reaction at all.
- If price later prints a **new** level at a slightly different price near an old one, mark the new level and downgrade (don't delete) the old one — give priority to the newer level if price returns to that general zone, but keep the old one as a lower-weight reference.
- Never fully delete old levels — keep them, just with reduced priority/visibility, since they can still matter as reference points or regain relevance via confluence later.

## Chart organization tip (the main practical addition here)
- Use TradingView's object tree to group lines into folders by category: daily levels, weekly levels, old/downgraded daily levels, Fibonacci pulls, high-volume nodes, trend lines, miscellaneous S&R, etc.
- Toggle whole folders on/off rather than managing individual lines — keeps the chart readable while still preserving less-current levels for occasional reference (e.g. briefly turning on "old daily levels" when price approaches a zone, to check if an older level might still apply).
